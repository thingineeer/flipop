# FLIPOP 게임 정체성 & 정책 문서

> 이 문서는 FLIPOP의 핵심 정체성, 설계 철학, 게임 정책을 정의합니다.
> 모든 기능 추가/변경은 이 문서의 원칙에 부합해야 합니다.

---

## 1. 게임 정체성 (Identity)

### 한 줄 정의
> **"탭 한 번으로 연쇄를 만드는, 순수 실력 기반 캐주얼 퍼즐"**

### 핵심 USP (Unique Selling Point)
FLIPOP의 "탭하면 인접 블록까지 색이 순환"하는 메커니즘은 **시장에 직접적 경쟁작이 없는 독자 메커니즘**이다.
- Columns(SEGA): 자기 블록 내부만 순환
- Candy Crush: 인접 블록과 스왑
- Puyo Puyo: 같은 색 4개 연결
- **FLIPOP: 탭한 블록의 상하좌우가 함께 변함 → "영향 범위의 확장"**

이 메커니즘이 만드는 핵심 경험:
1. **의도하지 않은 연쇄의 발견** (Discovery 미학)
2. **"한 수 앞"을 읽는 전략적 깊이** (Interesting Decisions)
3. **결정론적 행동 + 랜덤 보드 = 순수 실력** (Input Randomness Only)

### 감성 키워드
`귀여움` · `직관적` · `중독성` · `공정함` · `숙련의 쾌감`

## 2. 설계 철학 (Design Philosophy)

### 원칙 1: Elegance (우아함) > Features (기능 수)
```
Elegance = Depth / Complexity
```
- 규칙은 최소로, 전략적 깊이는 최대로
- 새 규칙 추가 시 반드시 검증: "이것이 Depth를 Complexity보다 더 올리는가?"
- 현재 핵심 규칙은 **3개뿐**: (1) 탭→인접 순환, (2) 같은 색 줄→클리어, (3) N턴마다 새 줄
- 참고: 바둑은 규칙 2개로 극한의 깊이 달성

### 원칙 2: Easy to Learn, Hard to Master
- **Skill Floor**: 30초 내 이해 가능 (탭하면 색이 바뀜 → 같은 색 줄 = POP)
- **Skill Ceiling**: 다단계 연쇄 설계, 색 순환 예측, 그리드 전체 조망
- 초보자와 숙련자의 점수 차이가 10배 이상 나야 한다

### 원칙 3: 행동은 결정론적, 보드만 랜덤 (Input Randomness Only)
- 플레이어의 탭 결과는 100% 예측 가능 (Output Randomness 없음)
- 랜덤은 게임 시작 보드 배치와 새 줄 추가에만 적용
- 이것이 "순수 실력 게임"의 근거이자 리더보드의 공정성 보장

### 원칙 4: 능동적 난이도 조절 (Active DDA)
- 외부에서 난이도를 강제하지 않음
- 그리드가 쌓이면 자연스럽게 압박 증가, 잘 클리어하면 여유 생성
- Jenova Chen의 Flow 채널 이론: 불안과 지루함 사이의 좁은 통로

### 원칙 5: 도파민 루프는 연쇄(Combo)에 집중
- B.F. Skinner의 Variable Ratio Reinforcement: 예측 불가한 연쇄가 도파민 분비
- 3단계 보상 루프: 탭(즉각 피드백) → 클리어(POP + 점수) → 연쇄(폭발적 보상)
- **보상 자체보다 "보상의 연출"이 루프를 강화**

## 3. 게임 정책 (Game Policies)

### 3.1 공정성 정책 (Fairness)
| 정책 | 상세 |
|------|------|
| **순수 실력 기반** | 게임플레이에 영향을 주는 유료/광고 보상 없음 |
| **점수 2배 금지** | 광고 시청으로 점수를 증폭하는 것은 리더보드 형평성에 반함 |
| **시간 연장 금지** | 광고로 타이머를 늘리면 동일 조건 경쟁이 무의미해짐 |
| **이어하기 금지** | 게임 오버 후 부활은 순수 실력 측정을 왜곡함 |
| **리더보드 무결성** | 같은 조건(시간, 색 수, 새 줄 빈도)에서 경쟁해야 의미 있음 |

### 3.2 수익화 정책 (Monetization)
| 정책 | 상세 |
|------|------|
| **인터스티셜 광고** | 게임 오버 직후 1회만. 플레이 중 절대 노출 안 함 |
| **리워드 광고** | 코스메틱(블록 테마, 배경) 해금에만 사용. 게임플레이 이점 없음 |
| **배너 광고 금지** | 화면 공간 침해 + 몰입 방해. 절대 사용 안 함 |
| **IAP** | 코스메틱 전용 (추후 도입 시). 게임플레이 이점 없음 |

### 3.3 게임 모드 정책
| 모드 | 설명 | 조건 |
|------|------|------|
| **Classic** | 90초 타이머, 3색, 3턴마다 새 줄 | 현재 기본 모드 |
| **Zen (무한)** | 타이머 없음, 그리드 오버플로만으로 게임 오버 | 연습/힐링용 |
| **Sprint** | 60초 타이머, 빠른 템포 | 짧은 세션 |
| **Progressive** | 점수에 따라 난이도 자동 상승 (색 추가, 줄 빈도 증가) | 도전 모드 |

### 3.4 클리어 메커니즘 정책
| 정책 | 상세 |
|------|------|
| **가로줄 클리어** | 기존 유지 — 5칸 모두 같은 색이면 POP |
| **세로줄 클리어** | 신규 추가 — 연속 3개 이상 같은 색이면 POP |
| **동시 클리어** | 가로+세로 동시 발생 시 보너스 포인트 |
| **연쇄 (Combo)** | 중력 낙하 후 새로운 클리어 자동 체크, 콤보 배율 적용 |

### 3.5 난이도 정책
| 구간 | 색 수 | 새 줄 빈도 | 비고 |
|------|-------|-----------|------|
| 0-499점 | 3색 | 3턴마다 | 입문 구간 |
| 500-999점 | 3색 | 2턴마다 | 압박 시작 |
| 1000점+ | 4색 | 2턴마다 | 본격 도전 |

### 3.6 새 줄 생성 정책 (Controlled Randomness)
- 기본: 완전 랜덤 색상 배치
- "거의 완성된 줄" (4/5 같은 색) 확률: ~15%
- 이유: 즉각적 목표를 제공하여 흥분감 유발 (Input Randomness의 질 향상)
- 단, 이 비율이 너무 높으면 난이도 하락 → 밸런싱 필요

## 4. 피드백 & 연출 정책 (Game Feel / Juice)

### 콤보별 연출 수준
| 콤보 | 시각 효과 | 촉각 (Haptic) | 시간 보너스 |
|------|----------|-------------|-----------|
| 1x | POP 파티클 | lightImpact | +3초 |
| 2x | 파티클 + 화면 번쩍임 | mediumImpact x2 | +5초 |
| 3x+ | 파티클 + 번쩍임 + 화면 흔들림 | heavyImpact | +7초 |
| 5x+ | 전체 화면 이펙트 + 무지개 연출 | heavyImpact x2 | +10초 |

### 위기 상황 연출
- 그리드 높이 5/6 이상: 배경색 점진적 붉은빛, 위험 라인 깜빡임
- 타이머 15초 이하: 타이머 붉은색 + 펄스 애니메이션

## 5. 콘텐츠 전략 (Long-term Engagement)

### Raph Koster 원칙 적용
> "Fun = Learning. Boredom = When the game stops teaching."

| 시기 | 학습 내용 | 유지 장치 |
|------|----------|----------|
| 첫 1분 | 탭 → 색 변환 → 줄 클리어 | 온보딩 오버레이 |
| 첫 10판 | 콤보 발견, 중력 활용 | 콤보 연출 피드백 |
| 첫 1주 | 세로줄 클리어 전략, 점수 패턴 최적화 | 리더보드 경쟁 |
| 이후 | 다단계 연쇄 설계, 4색 관리 마스터리 | Progressive 모드, 일일 챌린지 (추후) |

### "한 판 더" 루프 강화
- 게임 오버 → 점수 표시 → PLAY AGAIN 최소 마찰 (1탭)
- 짧은 세션 (60-120초)으로 시간 부담 없음
- "이번엔 이 전략으로..." 동기 자연 발생

## 6. 기존 대비 변경 요약

### 제거 항목
- [x] 점수 2배 광고 보상
- [x] 시간 +30초 광고 보상
- [x] 이어하기 광고 보상

### 추가 항목
- [ ] 세로줄 클리어 (연속 3개 이상 같은 색)
- [ ] Progressive 난이도 (점수 기반 색 추가 + 줄 빈도 증가)
- [ ] 콤보 5x+ 연출 강화
- [ ] 위기 상황 연출 (배경색, 타이머 펄스)

### 유지 항목
- 핵심 메커니즘 (탭 → 인접 색 순환)
- 인터스티셜 광고 (게임 오버 후)
- 리더보드 시스템
- 아바타/닉네임 시스템
- 4개 언어 지원

## 7. 이론적 근거 요약

| 이론/프레임워크 | 핵심 인사이트 | FLIPOP 적용 |
|---------------|-------------|------------|
| MDA Framework | Challenge+Submission 강, Discovery 약 | 연쇄 발견의 즐거움 강화 |
| Sid Meier "Interesting Decisions" | 트레이드오프가 결정의 무게감 | 즉시 클리어 vs 콤보 세팅 선택 |
| Depth vs Complexity | Elegance = Depth/Complexity 최대화 | 규칙 3개 유지, 세로줄은 기존 메커니즘의 자연 확장 |
| Variable Ratio Reinforcement | 예측 불가 보상이 도파민 극대화 | 연쇄 콤보 = 가변 비율 강화 |
| Emergent Gameplay | 간단한 규칙의 상호작용 → 복잡한 상황 출현 | 색순환+중력+세로클리어 조합 |
| Flow Theory (Csikszentmihalyi) | 도전-기술 균형의 좁은 채널 | 능동적 DDA (그리드 높이 = 난이도) |
| Tetris Effect | 반복 패턴 인식 → 몰입 | 3색 순환 읽기의 명상적 경험 |
| Raph Koster "Theory of Fun" | Fun = Learning, Boredom = 학습 중단 | Progressive 난이도로 지속적 학습 |
| Input vs Output Randomness | Input만 랜덤 → 실력 게임 | 보드만 랜덤, 행동은 결정론적 |
