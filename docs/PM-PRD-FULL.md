> Flutter 기반 모바일 캐주얼 퍼즐 게임

|---|------|------|
| **1** | **문제 (Problem)** | 1) 기존 캐주얼 퍼즐(매치3)은 수천 개 스테이지를 요구해 피로감 유발 2) 짧은 자투리 시간에 적합한 "한 판 완결형" 퍼즐 게임 부족 3) 경쟁 요소(순위표)가 없거나 과금 벽 뒤에 숨겨진 게임이 대부분 |

## 3. PRD (Product Requirements Document)

### 3.1 제품 비전 & 미션

**비전**: "전 세계 누구나 30초 만에 배우고, 3분 안에 빠져드는 색 순환 퍼즐의 대표 게임"

**미션**: 단순한 탭 조작으로 깊은 전략성과 시각적 쾌감을 동시에 제공한다. 귀여운 캐릭터와 POP 이펙트로 감성을 자극하고, 글로벌 순위표로 건전한 경쟁 문화를 만든다.

**제품 원칙**:
1. **즉시 플레이**: 회원가입 없이 익명 인증으로 바로 게임 시작
2. **1분 학습, 무한 도전**: 온보딩 3스텝(10초) 후 누구나 플레이 가능
3. **감각 보상**: 매 클리어마다 시각(파티클) + 촉각(햅틱) + 청각(효과음) 피드백
4. **공정한 경쟁**: 과금으로 점수를 살 수 없음. 순수 실력 기반 순위표
5. **존중하는 광고**: 플레이 중 절대 광고 없음. 게임오버 후에만 노출

### 3.2 타겟 유저

| 세그먼트 | 설명 | 비중 |
|----------|------|------|
| **코어 타겟** | 10~30대 모바일 캐주얼 게이머. 출퇴근/등하교 시 자투리 시간 활용. 짧은 세션, 높은 빈도 | 60% |
| **확장 타겟** | 30~50대 매치3/퍼즐 게임 경험자. 직관적 UI와 귀여운 비주얼에 반응 | 25% |
| **니치 타겟** | 퍼즐 하드코어 유저. 연쇄 콤보 최적화, 순위표 최상위 경쟁 | 15% |

**사용 환경**:
- 이동 중 (지하철, 버스) -- 한 손 조작, 세로 모드 고정
- 대기 시간 (카페, 병원, 학교 쉬는 시간) -- 1~3분 세션
- 취침 전 리라이닝 -- 5~10분 연속 플레이

### 3.3 핵심 기능 목록

#### 현재 구현 완료 (v1.0.0)

| ID | 기능 | 설명 | 상태 |
|----|------|------|------|
| F-001 | **5x7 그리드 퍼즐** | 5열 x 7행(가시 영역 6행 + 오버플로 1행) 그리드. 초기 3줄 랜덤 블록 배치 | 완료 |
| F-002 | **탭 -> 인접 색 순환** | 블록 탭 시 상하좌우 4방향 인접 블록의 색이 다음 색으로 순환 (빨강->파랑->노랑->빨강) | 완료 |

| F-004 | **중력 시스템** | 클리어 후 빈 칸 위의 블록이 아래로 낙하 | 완료 |

| F-006 | **새 줄 추가** | N턴(기본 3턴)마다 바닥에 랜덤 새 줄 추가. 기존 블록 1칸 위로 밀림 | 완료 |
| F-007 | **게임오버** | 블록이 7행(index 6)에 도달하면 게임오버 | 완료 |
| F-008 | **4종 픽셀 캐릭터** | 고양이(빨강), 강아지(파랑), 토끼(노랑), 개구리(초록) 픽셀 아트 블록 | 완료 |
| F-009 | **POP 파티클 이펙트** | 블록 클리어 시 6개 파티클 폭발 + 반짝임 효과 | 완료 |

### 3.5 비기능 요구사항

#### 성능

| 항목 | 목표 | 현재 상태 |
|------|------|-----------|
| 앱 콜드 스타트 | < 2초 | 측정 필요 |
| 블록 탭 응답 | < 16ms (60fps 유지) | 달성 |
| 파티클 렌더링 | 동시 30개 파티클, 60fps 유지 | 달성 |
| 순위표 로딩 | < 1초 (50건 기준) | 달성 (Firebase 인덱싱) |
| APK/IPA 크기 | < 30MB | 측정 필요 |
| 메모리 사용량 | < 150MB | 측정 필요 |

#### 보안

| 항목 | 설명 |
|------|------|
| 인증 | Firebase Anonymous Auth -> Google/Apple Sign-In 업그레이드 경로 확보 |
| 데이터 접근 | Firestore Security Rules: 본인 문서만 쓰기, 순위표는 읽기 전용 |
| 점수 위변조 방지 | v1.0은 클라이언트 제출 (MVP). v1.1에서 Cloud Functions 서버 검증 도입 예정 |
| API 키 | firebase_options.dart는 git에 포함하되, Firestore Rules로 접근 제한 |

#### 접근성

| 항목 | 설명 |
|------|------|
| 색각 이상 대응 | 4색 블록에 캐릭터 이미지(고양이/강아지/토끼/개구리) 병용 -> 색상만으로 구분하지 않아도 됨 |
| 텍스트 크기 | 최소 11sp, 주요 점수 32sp 이상 |
| 터치 타겟 | 블록 크기 최소 44x44pt (셀 사이즈 동적 계산) |
| 화면 방향 | 세로 모드 고정 (portraitUp) |
| 언어 | 현재 한국어. v1.1에서 영어/일본어 다국어 지원 예정 |
| 다크 모드 | 현재 미지원. v1.2에서 다크 테마 추가 예정 |
